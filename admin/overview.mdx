## AccountStatus state machine

Every `Child` record carries a `status` field that moves through the following states. Transitions are triggered by admin actions and never by the student directly.

| Status | Meaning | How to advance |
|---|---|---|
| `pending_payment` | Student registered; payment not yet submitted or pending review. | Admin approves the payment. |
| `approved_pending_login` | Payment approved; login credentials not yet created. | Admin creates username + password in the Approved Users screen. |
| `assessment_required` | Credentials created; student must complete the initial assessment. | Student submits the assessment. |
| `pending_level_review` | Assessment submitted; admin must review and assign a level. | Admin assigns a `LiteracyLevel` via the Assessments screen. |
| `active` | Level assigned; student has full access to daily tasks. | â€” (normal operating state) |
| `rejected` | Payment was rejected. Student cannot proceed. | â€” (terminal state) |

## Audit log

Every significant admin action writes a row to `AdminAuditLog`. The `action` field is one of the following enum values:

| AuditAction | Triggered when |
|---|---|
| `PAYMENT_APPROVED` | Admin approves a pending payment. |
| `PAYMENT_REJECTED` | Admin rejects a pending payment. |
| `CREDENTIALS_CREATED` | Admin sets a student's username and password. |
| `LEVEL_ASSIGNED` | Admin assigns a `LiteracyLevel` after reviewing an initial assessment. |
| `LEVEL_CHANGED` | Admin changes a student's level after it was already assigned. |
| `DAILY_SKILL_FOCUS_SET` | A daily task's skill focus is configured. |
| `CONTENT_CREATED` | A new content item is added to the library. |
| `CONTENT_ASSIGNED` | A content item is linked to a daily task. |
| `STUDENT_PROFILE_EDITED` | Admin updates a student's or parent's profile fields. |

Each log row links back to the acting admin and up to one target entity (child, payment, content item, daily task, or assessment) through nullable foreign keys.
